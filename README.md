# 플로우 SaaS 개발자 부문 과제 

## 과제 목표 
- 파일 확장자 차단 프로그램 

## 과제 배경 
- 어떤 파일들은 첨부시 보안에 문제가 될 수 있습니다.. 특히 exe, sh 등의 실행파일이 존재할 경우 서버에올려서 실행이 될 수 있는 위험이 있어 파일 확장자 차단을 하게 되었습니다. 

## 과제 기본 기능 정의 
- 고정 확장자 : 자주 차단하는 확장자 (bat, cmd , com , cpl , exe , scr , js)
    - 고정확장자는 check or uncheck 기능이 존재 default 는 uncheck 
    - 상태값 변경시 DB 에 저장되며 새로고침 시 유지 
- 커스텀 확장자 : 고정 확장자 이외의 추가할 확장자  
    - 최대 20자 까지 등록 가능 
    - 최대 200개 까지 등록 가능
    - 등록 시 DB 에 저장
    - 삭제 시 DB 에서 삭제  

## 제공 요건 외의 추가 사항 
- 커스텀 확장자의 등록시 영어 소문자 , 숫자 만 입력이 가능. 단 숫자만 입력할 수 없음.
    - 다중 확장자 입력 방지 
    - "EXE" , "Exe" 등 실행파일의 경우 확장자의 대소문자 구분없이 실행 가능성이 있으므로 소문자로만 등록 후 실제 파일 업로드시 **LowCase** 로 벨리데이션 체크를 진행해야 보안상 이슈가 없을것으로 생각됨
- 커스텀 확장자 등록시 고정 확장자와 , 커스텀 확장자에 중복 등록 되지 않도록 차단 
    - 이미 존재하는 데이터가 있으므로 중복 저장하여 불필요한 데이터가 생기지 않도록 고려함 
    - 백엔드와 DB 모두 제약을 주어 이중 보완 처리
